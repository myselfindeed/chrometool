浏览器插件需求说明文档
根据要求写一个谷歌浏览器自用插件，需适配苹果M2芯片。
一、插件概述

插件名称：Web元素数据采集器  
功能定位：帮助用户快速从网页中提取指定元素内容，并导出为结构化Excel文件  
目标用户：需要从网页批量采集数据的内容编辑、数据分析师、市场研究人员等

二、核心功能

1. 元素选取与排序

• 选取范围：支持选择1-10个网页元素

• 选取方式：鼠标点击/悬停高亮显示可选取元素

• 顺序调整：已选元素支持拖拽上下排序

• 别名设置：为每个元素设置自定义名称（作为Excel列名）

2. 内容采集

• 按序采集：按照用户设定的元素顺序获取内容

• 内容类型：

  • 文本内容（innerText）

  • 链接地址（href属性）

  • 图片地址（src属性）

  • 自定义属性值

• 多页采集：支持分页场景下自动翻页采集（可选功能）

3. 数据导出

• Excel生成：将采集内容生成标准Excel文件

• 列名映射：Excel列名与元素别名一致

• 本地保存：支持选择保存路径到电脑本地

• 文件命名：默认"数据采集_日期时间.xlsx"，支持自定义

三、界面设计

1. 整体布局

• 位置：浏览器右侧悬浮面板

• 宽度：屏幕宽度的30%（约400-500px）

• 高度：与浏览器窗口同高

• 交互方式：可展开/折叠，默认展开

2. 视觉风格

• 设计风格：简洁扁平化设计

• 配色方案：

  • 主色：#4285F4（Google蓝）

  • 辅色：#34A853（Google绿）

  • 背景：白色/浅灰

• 图标：使用Material Design风格图标

四、交互流程

1. 正常流程


打开插件 → 点击"开始选取" → 在页面选择元素 → 设置别名 → 排序 →
点击"开始采集" → 查看采集结果 → 确认列名 → 点击"导出Excel" → 保存文件


2. 异常处理

• 未选择元素：点击采集按钮时提示"请先选择至少1个元素"

• 采集失败：显示具体失败原因（元素不存在、网络错误等）

• 导出失败：提示失败原因并提供重试选项

五、技术实现要点

1. 前端技术

• 框架：Vue.js 3 + TypeScript

• UI组件库：Element Plus 或 Vuetify

• 拖拽排序：使用Sortable.js或Vue Draggable

2. 浏览器API

• 内容脚本：通过content script与页面交互

• 元素选取：使用document.querySelector等DOM API

• 消息通信：chrome.runtime.sendMessage实现前后端通信

3. Excel生成

• 库选择：SheetJS (xlsx)

• 文件保存：使用chrome.downloads API

4. 数据存储

• 配置存储：chrome.storage.sync保存用户设置

• 临时数据：chrome.storage.local存储采集结果


六、注意事项

1. 权限申请：需要声明以下权限
   • "activeTab" - 获取当前标签页内容

   • "downloads" - 保存文件到本地

   • "storage" - 存储用户设置

2. 性能优化：大量数据采集时注意分批次处理，避免页面卡顿

3. 用户体验：提供清晰的反馈和引导，降低使用门槛

4. 隐私保护：明确声明数据仅本地处理，不上传服务器
